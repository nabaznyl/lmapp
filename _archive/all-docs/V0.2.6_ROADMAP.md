# v0.2.6 Roadmap (Internal Planning)

**Status:** Planning phase  
**Target Release:** December 25, 2025  
**Focus:** Plugin ecosystem expansion + performance + voice I/O  

## Phase 1: Plugin Expansion (5-7 New Production Plugins)

### Tier 1 - High-Value Core Plugins (2)

1. **Dependency Auditor Plugin**
   - Scans project dependencies (Python, Node, etc.)
   - Checks for vulnerabilities and license compliance
   - Integrates with CVE databases
   - Estimated: 300 LOC, 20 tests
   - Competitive advantage: Offline + privacy-first
   - Use case: "Audit project dependencies without cloud"

2. **Cache Manager Plugin**
   - LLM response caching (reduce API calls)
   - RAG document cache optimization
   - Cache invalidation strategies
   - Estimated: 250 LOC, 18 tests
   - Use case: "Faster responses, lower cost"

### Tier 2 - Differentiation Plugins (3-4)

3. **Document Chatbot Plugin**
   - Enhanced version of Q&A bot
   - Multi-document conversation context
   - Citation tracking (which doc answered this?)
   - Estimated: 280 LOC, 22 tests
   - Use case: "AI search engine for local docs"

4. **Code Refactoring Assistant**
   - Pattern detection and suggestions
   - Auto-refactoring with safety checks
   - Before/after diffs
   - Estimated: 350 LOC, 25 tests
   - Use case: "GitHub Copilot for refactoring"

5. **Knowledge Base Builder**
   - Convert notes/articles to searchable KB
   - Auto-tagging and categorization
   - Query expansion (synonyms, related topics)
   - Estimated: 400 LOC, 28 tests
   - Use case: "Personal knowledge graph"

6. **Research Assistant Plugin** (Optional)
   - Paper/article summarization
   - Citation management
   - Key findings extraction
   - Estimated: 320 LOC, 24 tests
   - Use case: "Research accelerator"

## Phase 2: RAG Performance Improvements

### Priority 1 - Search Quality (No ML deps)
- Better tokenization (handle punctuation, contractions)
- TF-IDF weighting (improve relevance ranking)
- Keyword extraction from queries
- Estimated effort: 100 LOC, 12 tests
- Impact: 30% better search accuracy

### Priority 2 - Caching Strategy
- Cache search results (LRU, TTL-based)
- Batch indexing (faster multi-file processing)
- Lazy loading of large documents
- Estimated effort: 150 LOC, 15 tests
- Impact: 50% faster searches on repeated queries

### Priority 3 - Scale Testing
- Test with 10k+ documents
- Memory profiling and optimization
- Chunking strategy for large files
- Estimated effort: 80 LOC, 10 tests
- Impact: Handle real-world document volumes

## Phase 3: Voice I/O Integration

### Optional Feature - Voice Search
- Speech-to-text input (pyttsx3 or system integration)
- Text-to-speech output (espeak)
- Voice command parsing
- Plugin approach (don't break core)
- Estimated effort: 200 LOC, 12 tests
- Dependencies: pyttsx3 (optional), espeak (system)
- Status: Keep local-only until finalized

## Phase 4: Web UI - GitHub Copilot-Style Browser Interface

### Optional Built-In Web Server Component (High Priority)
**Purpose:** Self-hosted web interface for LMAPP (alternative to CLI)

**Design Inspiration:** GitHub Copilot website aesthetic
- Dark theme (dark_high_contrast style similar to GitHub)
- Minimal, clean interface
- Professional typography and spacing
- Responsive design
- Real-time chat interface

**Components:**

1. **Web Server Module** (`src/lmapp/web/server.py`)
   - FastAPI backend serving web UI
   - REST API endpoints for chat, documents, plugins
   - WebSocket support for real-time chat
   - Static file serving (HTML/CSS/JS)
   - Estimated: 300 LOC

2. **Frontend** (`src/lmapp/web/static/`)
   - Pure HTML/CSS/JavaScript (no build step)
   - GitHub Copilot-inspired dark theme
   - Chat interface (similar to Copilot chat)
   - Document management view
   - Plugin marketplace browser
   - Settings/configuration panel
   - Estimated: 800+ LOC

3. **Features:**
   - Real-time chat with streaming responses
   - File/document upload
   - Citation tracking (show sources)
   - Plugin browser and installer
   - Conversation history
   - Dark mode toggle
   - Keyboard shortcuts
   - Export conversations
   - Responsive mobile view

4. **Architecture:**
   - Lightweight (no npm/webpack needed)
   - Works offline (serves static files)
   - Optional - doesn't replace CLI
   - Can run on any port (default 5000)
   - Serves at `http://localhost:5000`

5. **Estimated Effort:**
   - Web server: 2-3 hours (300 LOC)
   - Frontend: 4-5 hours (800+ LOC)
   - Tests: 1-2 hours (25-30 tests)
   - Total: 7-10 hours (1,100+ LOC)

6. **Success Metrics:**
   - Web UI launches successfully
   - Chat works with backend
   - Document upload functional
   - Plugin marketplace browsable
   - Mobile responsive
   - Dark theme matches GitHub Copilot aesthetic

### Integration Points

## Strategic Value

**Copilot Alternative Vision:**
- 5 new plugins = 5 new use cases (search, audit, refactor, knowledge, chat)
- Voice I/O = hands-free operation
- RAG improvements = production-ready for real docs

**AI Search Engine Vision:**
- Document Chatbot = "Google for your local docs"
- Knowledge Base Builder = "Personal knowledge graph"
- Better RAG = "Google-quality search offline"

**Competitive Advantage:**
- Dependency Auditor = privacy-first, no cloud
- Cache Manager = offline optimization
- Voice I/O = differentiation from web-based alternatives

## Risk Mitigation

- **Plugin bloat:** Keep each <400 LOC, follow BasePlugin pattern
- **Performance regression:** Benchmark RAG before/after changes
- **Scope creep:** Freeze features by Dec 24 for release
- **Community overwhelm:** Keep plugin docs simple, provide templates
