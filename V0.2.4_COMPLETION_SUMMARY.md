### RAG System (Retrieval-Augmented Generation)
**Purpose:** Semantic search over local files for context injection  
**Key Classes:**
- `Document` - Content wrapper with metadata
- `SimpleVectorizer` - Jaccard-based similarity scoring

- `RAGSystem` - High-level interface

**Capabilities:**
- Index text, code, markdown, JSON, HTML files
- Search by semantic similarity (no ML needed)
- Extract context for LLM prompts
- Full persistence to disk

**Test Coverage:** 16 tests

**Purpose:** Extend LMAPP with custom functionality

- `PluginInfo` - Runtime state tracking
- `PluginManager` - Discovery and lifecycle management

- Dynamic module loading with error isolation

## ðŸ“š Architecture Decisions

### 1. **No External ML Dependencies**
- **Why:** Keep installation lightweight, avoid complex setup
- **How:** Jaccard similarity for semantic search (pure Python)
- **Trade-off:** Lexical search vs. semantic (acceptable for local use)

### 2. **JSON-Based Persistence**
- **Why:** Human-readable, works with standard tools (grep, jq, etc.)
- **How:** All classes implement to_dict() and from_dict()
- **Benefit:** Easy debugging and inspection

### 3. **Plugin Architecture**
- **Why:** Enable community contributions without core changes
- **How:** BasePlugin abstract class with standardized interface
- **Benefit:** Extensible without modifying lmapp core

### 4. **Batch Processing**
- **Why:** Handle multiple inputs efficiently
- **How:** Job-based processing with persistence
- **Benefit:** Progress tracking, result export, job resumption
